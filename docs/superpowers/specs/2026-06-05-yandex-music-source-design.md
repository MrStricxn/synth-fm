# Подключение неофициального API Яндекс.Музыки + временное отключение прочих источников

**Дата:** 2026-06-05
**Статус:** утверждён, готов к плану реализации

## Цель

Добавить Яндекс.Музыку как источник воспроизведения в synth-fm и **временно** (обратимо) отключить
существующие источники Audius и hearthis, чтобы протестировать качество выдачи Яндекса.
Источники не удаляем — переключение делается одним флагом.

## Контекст

Текущие источники (`src/api/audius.js`, `src/api/hearthis.js`) — открытые, без ключей, с CORS `*`,
отдают прямой `audio/mpeg`. `usePlayerStore.js` агрегирует их нормализованные треки в 4 местах:
лента (feed), поиск (search), рекомендации (recommendations), seed-артисты. Воспроизведение —
нативный `<audio>` в `AudioEngine.jsx`, ожидает поле `streamUrl`.

Яндекс принципиально отличается: **нет CORS** и **нужен OAuth-токен** аккаунта (есть подписка Plus →
полные треки). Прямые вызовы из браузера невозможны.

## Архитектура

### Прокси (снятие CORS)

Используем `acherkashin/yandex-music-cors-proxy` — тонкую обёртку над `cors-anywhere`. Прокси
прозрачно пробрасывает запрос на `api.music.yandex.net` и добавляет CORS-заголовки. Вся
Яндекс-логика живёт во фронте; прокси снимает CORS только с JSON-запросов.

- **Дефолт:** локальный `cors-anywhere` на `localhost:8080` (токен не утекает третьей стороне).
- Адрес прокси — `VITE_YANDEX_PROXY` (`.env.local`, дефолт `http://localhost:8080`).
- Запуск отдельным npm-скриптом (`npm run proxy`).
- Финальный mp3 — подписанная ссылка на хранилище Яндекса; играется напрямую в `<audio>`
  без токена и без прокси.

### Токен

- `VITE_YANDEX_TOKEN` в `.env.local` (добавить `.env.local` в `.gitignore`; в спеку/коммиты не пишем).
- В браузере прикрепляется как заголовок `Authorization: OAuth <token>` к JSON-запросам Яндекса.
- Получение: OAuth-ссылка официального приложения
  `https://oauth.yandex.ru/authorize?response_type=token&client_id=23cabbbdc6cd418abb4b39c32c41195d`
  → после логина токен в адресной строке в `#access_token=...`.

### Клиент `src/api/yandex.js`

В стиле существующих `audius.js` / `hearthis.js`. Базовый URL запроса:
`${VITE_YANDEX_PROXY}/https://api.music.yandex.net/<endpoint>`, заголовок `Authorization`.

- `searchTracks(query, limit)` → `/search?text=<q>&type=track&page=0` → массив треков.
- `getStreamUrl(trackId)`:
  1. `GET /tracks/<id>/download-info` (auth) → список вариантов `{codec, bitrate, downloadInfoUrl}`;
     выбрать mp3 наибольшего битрейта.
  2. `GET <downloadInfoUrl>&format=json` (через прокси) → `{host, path, ts, s}`.
  3. `sign = md5("XGRlBW9FXlekgbPrRHuSiA" + path.slice(1) + s)`.
  4. `streamUrl = https://<host>/get-mp3/<sign>/<ts><path>`.
- `normalizeTrack(t)` → форма, общая для всех источников:
  `{ id: 'ya_<id>', title, artist, artwork, streamUrl, duration(ms), genre, playCount, color, source: 'yandex' }`.
  Артворк Яндекса приходит шаблоном `…/%%` → подставить `400x400`. `color` — через `colorFor` из `audius.js`.
  `streamUrl` резолвится лениво (по запросу воспроизведения) либо предзагрузкой — см. ниже.
- `trendingTracks()` → чарт `/landing3/chart` (или плейлист «Дежурный»); нормализация та же.
- `resolveSeed(terms, perTerm)` → как у других источников (поиск по seed-артистам, дедуп).

**Резолв streamUrl:** download-info требует отдельного запроса на трек. Чтобы не делать N запросов
на всю ленту, `streamUrl` резолвим лениво в момент проигрывания. Минимальный путь: в `normalizeTrack`
кладём `trackId`, а перед воспроизведением (в store/AudioEngine) вызываем `getStreamUrl`. Если это
потребует правки точки старта воспроизведения — внести точечно, поведение `<audio>` не меняем.

### Временное отключение Audius / hearthis

В `usePlayerStore.js` — флаг источника (напр. `const SOURCE = 'yandex'` или
`VITE_PLAYER_SOURCE`). В 4 точках агрегации (feed/search/recommendations/seed) при `yandex`
ходить только в `yandex.js`. Импорты и вызовы audius/hearthis **оставить** — просто шунтировать,
чтобы откат = смена флага. Ничего не удалять.

### Воспроизведение

`AudioEngine.jsx` без изменений — формат `streamUrl` (прямой mp3) тот же, что у Audius/hearthis.

## Обработка ошибок

- Прокси недоступен / 502 → как у существующих источников: `.catch(() => [])`, лента не падает.
- Нет/протух токен (401) → понятное сообщение в UI/консоль с подсказкой обновить `VITE_YANDEX_TOKEN`.
- Трек без mp3-варианта в download-info → пропустить (как `playable`-фильтр в hearthis).

## Тестирование

- Юнит: `normalizeTrack` (шаблон артворка `%%`→`400x400`, ms-длительность, форма объекта).
- Юнит: сборка подписи — детерминированный вход (`path`, `ts`, `s`) → известный `sign`/URL.
- Существующие тесты store/компонентов должны остаться зелёными (флаг не ломает контракты).

## Вне области (YAGNI)

- Деплой прокси в прод, продовый секрет-стор.
- UI-поле ввода токена (выбран `.env.local`).
- Лайки/плейлисты Яндекса, синхронизация библиотеки Яндекса.

## Безопасность

- `.env.local` в `.gitignore`; токен не коммитим и не пишем в спеку.
- Локальный прокси по умолчанию — токен не уходит третьим лицам.
- Токен, засветившийся в переписке, рекомендуется перевыпустить после тестов.
